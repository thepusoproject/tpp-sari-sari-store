Learning path snapshot

Mastery-based progression with one evolving case study: the TPP Sari-Sari Store CLI.

01 · Clone the template

Use the TPP Sari-Sari Store template and name it tpp-sari-sari-store-<yourname>. Keep it public so CI + reviews work.

02 · Build + run locally

Create a virtualenv, install requirements, and run python main.py. Tests live under tests/ for each module.

03 · Submit modules via PR

One branch per module. Push, open a PR, wait for green checks, then request review. Unlock the next module at ≥80%.

Enroll in the roster

Fill out the mini form—we’ll open a pre-filled GitHub issue so you just review and submit.

How it works

  1. Create your repo from the template (public) and invite thepusoproject as collaborator.
  2. Use the form to generate a roster issue. We pre-fill the title, label, and body for you.
  3. Review the GitHub issue that opens in a new tab and click Submit new issue.
  4. Give us a beat—the Gradeboard refreshes soon after we merge you into the roster.

Need help? Ping us before filing so we can keep the roster tidy.
