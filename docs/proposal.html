Education 5.0 Roadmap

This roadmap mirrors the Notion proposal: we start with the Programming para sa Lahat CLI pilot (Milestone 0) and layer in adaptive learning, wellbeing dashboards, and immersive/IoT pilots over time. Each phase keeps the “human + AI” guardrails front and center.

Milestones

Now shipping

Milestone 0 — CLI + Instructor Automations

  • Programming para sa Lahat curriculum + sari-sari-store project.
  • Automated grading, Gradeboard, concept prompts, oral-check workflow.
  • Cloudflare roster worker + GitHub-only submission pipeline.

Purpose: prove the coding-first stack, collect mastery data, and surface insights for Education 5.0 layers.

Milestone 1 — Adaptive & Wellbeing Layer

  • Rules-based “next best activity” feed with teacher overrides.
  • Wellbeing check-ins, mentoring notes, “needs attention” flags.
  • Studio authoring UX + ALS-aligned portfolio exports.

Matches proposal pillars 1 & 2: AI-augmented learning paths + holistic learner dashboard with privacy guardrails.

Milestone 2 — Immersive Projects & Offline PWA

  • VR/AR mini-simulations (2–5 modules) tied to Studios.
  • Offline-first PWA (draft + sync) for low-connectivity learners.
  • Community/values evidence capture (solidarity, creativity, integrity).

Matches pillar 3 + CTL IPOO: experiential learning with reflection + values baked in.

Milestone 3 — Smart Spaces Pilot (Optional)

  • Small IoT deployment (1–2 learning hubs) for environment + engagement data.
  • Strict consent + governance, only after LMS impact is proven.
  • Feedback loop into wellbeing dashboard + program decisions.

Matches pillar 4: connect physical spaces without compromising privacy.

Timeline at a glance

Acceptance checks per proposal

  • AI recommendations remain explainable + overrideable.
  • Wellbeing data has role-based visibility and opt-in consent.
  • Each Studio yields portfolio artefacts, reflections, and values evidence.
  • Offline patterns and low-bandwidth UX are considered in every milestone.
