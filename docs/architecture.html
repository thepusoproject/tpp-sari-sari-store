System Flow & Technology Stack

Taglish Py4E uses a lightweight but auditable pipeline: students code locally, GitHub Actions handles grading + reporting, and a Cloudflare Worker keeps roster forms inside the site. Below is the bird's-eye view.

  1. Students clone the template repo, code per module, and open a PR with concept answers + walkthrough.
  2. Module Grade workflow runs pytest + concept checks → updates grades/latest.json and the Gradeboard.
  3. Concept question bot comments on each PR with random theory prompts to be answered inline.
  4. Cloudflare Worker handles roster enrollment forms, creates GitHub issues, and hands them off to the Gradeboard action.
  5. GitHub Pages serves the Gradeboard + docs so teachers see live progress.

Core components

Student repo (GitHub)

CLI case study + modules, concept prompts, recordings folder. Students push PRs per module.

GitHub Actions

Module grading, concept validation, random question bot, oral-check reminders, roster gradeboard sync.

Cloudflare Worker

Receives roster form submissions from the site, creates GitHub issues securely (token never hits the browser).

GitHub Pages

Hosts the public Gradeboard, architecture doc, and proposal roadmap so stakeholders see progress.
