# Module 01 Playbook — Sukli Calculator

## 1. Purpose (CTL / IPOO mapping)
- **Input:** Surface why sukli accuracy matters; learners capture a real tindahan scenario (item, price, qty, actual cash tendered).
- **Process:** Provide a Taglish explainer + DIWA micro-lesson, then a “quest” where they interview or observe a tindera and log one real transaction.
- **Output:** Repo holds (a) coding task (`main.py`), (b) written reflection (`CONCEPT.md`), (c) quest evidence (`modules/module_01/STUDIO.json`).
- **Outcome:** Emphasize Solidarity (fair change = trust), Integrity (accurate logs), Creativity (turning manual steps into automation).

## 2. Studio assets (delivered through DIWA or SMS)
1. **Explain It (1–2 min video script)**  
   “Kahit maliit ang sukli, doon nabubuo ang tiwala. Sa lesson na ’to, gagawa tayo ng sukli calculator para ang tindera ay siguradong tama ang sinisingil at binibigay. Panoorin ang aktwal na resibo, bilangin ang sukli, at pag-usapan kung bakit kailangang i-cast sa tamang type ang presyo at dami.”

2. **Mini-check (choose one)**  
   *Prompt:* “Kung ang presyo ay `12.50` at quantity ay `3`, ano ang tamang data type sa Python?”  
   - A. `string`  
   - **B. `float`** ✅  
   - C. `bool`

3. **Quest instructions**  
   “Pumili ng isang totoong item sa tindahan. Tanungin ang presyong bentahan at ilista kung magkano ang binayad at ang sukli. Ipadala ang datos (item, presyo, qty, cash, actual sukli) sa DIWA. I-reflect kung saan madalas nagkakamali ang sukli.”

## 3. Delivery flow (before GitHub work)
1. **Day 0:** Learner receives the video + mini-check via DIWA (web/app/SMS). Must pass the mini-check to unlock the quest.  
2. **Day 1:** DIWA records quest answers and stores them as JSON:  
   ```json
   {
     "module": "M01",
     "item": "Lucky Me Pancit",
     "price": 12.50,
     "qty": 3,
     "cash": 100,
     "actual_change": 62.50,
     "insight": "Nagkamali dahil string concat ang nangyari sa lumang sheet"
   }
   ```
   This payload is synced to `modules/module_01/STUDIO.json` in the repo (either via PR bot or instructions to copy/paste).
3. **Day 2:** Learner clones the repo, copies the DIWA payload (if not auto-synced), and proceeds with coding + concept reflection.

## 4. Repo touchpoints
- `modules/module_01/README.md` should reference: “Complete the DIWA Studio first. Once you have the JSON snippet, place it in `STUDIO.json`.” (README update pending.)
- `modules/module_01/CONCEPT.md` holds the Taglish Q&A already in place.
- `modules/module_01/STUDIO.json` (new file, optional) stores the quest evidence.
- `modules/module_01/FEEDBACK.md` will be generated by the mentor workflow after the PR review.

## 5. Mentor + AI workflow
1. Mentor reviews quest evidence + concept answers.  
2. Teacher-only workflow (or manual script) runs the feedback helper → writes `FEEDBACK.md` → posts PR comment.  
3. Gradeboard pulls `grades/latest.json` + (future) Studio completion data to display mastery + engagement.
